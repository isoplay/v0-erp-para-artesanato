# Indice de documentacao

Use esta pasta como base do Obsidian para a documentacao do projeto.

## Leitura recomendada

1. [PRE-REQUISITOS.md](PRE-REQUISITOS.md) - ambiente necessario para rodar o sistema.
2. [SETUP.md](SETUP.md) - passos de instalacao e configuracao.
3. [ESTRUTURA.md](ESTRUTURA.md) - mapa geral das pastas e arquivos.
4. [CHANGELIST.md](CHANGELIST.md) - historico das alteracoes feitas no projeto.
5. [TESTES.md](TESTES.md) - roteiro de validacao manual.
6. [TROUBLESHOOTING.md](TROUBLESHOOTING.md) - erros comuns e solucoes.

## Observacoes

- O `README.md` principal permanece na raiz para o GitHub.
- A pasta `.obsidian/` fica apenas localmente e esta ignorada pelo Git.
- Arquivos gerados, como `.next/`, relatorios de teste e `*.tsbuildinfo`, nao devem ser versionados.
